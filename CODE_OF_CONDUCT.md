# Code of Conduct

## Our Pledge

We want AirHealth Monitor to be a respectful, inclusive, and productive project for everyone who contributes, reviews, tests, or discusses the work.

Project members and contributors are expected to communicate with care, assume good intent, and focus feedback on the work rather than the person.

## Expected Behavior

Examples of behavior that helps this project:

- Use welcoming and respectful language.
- Give constructive feedback with clear technical reasoning.
- Accept feedback gracefully and improve the work where appropriate.
- Keep discussions focused on the project goals, data quality, user safety, and software reliability.
- Respect different backgrounds, experience levels, and working styles.
- Be careful when discussing health-related features. Avoid presenting the project as a medical diagnosis tool.
- Protect secrets and private information, including API keys, database credentials, personal data, and sensor deployment details.

## Unacceptable Behavior

Examples of behavior that is not acceptable:

- Harassment, insults, intimidation, or personal attacks.
- Discriminatory language or behavior.
- Sexualized language or unwanted attention.
- Publishing private information without permission.
- Sharing secrets such as API keys, credentials, tokens, or private URLs.
- Deliberately introducing malicious code, data corruption, or misleading results.
- Misrepresenting health guidance as professional medical diagnosis or treatment.
- Repeatedly disrupting project discussions, reviews, or collaboration.

## Health and Safety Context

AirHealth Monitor provides environmental monitoring and first-care guidance. It should not be used as a replacement for professional medical advice.

Contributors should keep user safety in mind when changing health-risk logic, AI prompts, recommendations, dashboards, or documentation. When symptoms are severe or rapidly worsening, the product should direct users to seek qualified medical help.

## Reporting Issues

If you experience or observe behavior that violates this Code of Conduct, report it to the project maintainers:

If this repository is hosted on a platform such as GitHub, you may also use the platform's reporting or moderation tools.

Reports should include enough detail for maintainers to understand what happened, while avoiding unnecessary public exposure of private information.

## Enforcement

Maintainers may take action when this Code of Conduct is violated. Possible actions include:

- A private reminder or clarification.
- A request to edit or remove inappropriate content.
- Temporary restriction from project discussions or contributions.
- Permanent removal from project participation in serious or repeated cases.

Maintainers should apply enforcement fairly, consistently, and with attention to context and impact.

## Scope

This Code of Conduct applies in project spaces, including repository issues, pull requests, commits, documentation, discussions, chat channels, demos, and presentations related to AirHealth Monitor.

It also applies when someone represents the project in public spaces.
